# claude\-hunter 需求功能计划书

# 一、文档概述

本文档是 \`claude\-hunter\` 工具的完整需求功能计划书，整合了工具开发的背景、核心需求、功能细节、实现思路、优化建议及使用说明，全面覆盖前期所有讨论内容，作为工具开发、测试及使用的核心参考依据。

工具名称：claude\-hunter（ Claude Code CLI 超集可视化工具）

核心定位：基于 Mac 系统，兼容 Claude Code CLI 所有原生功能，同时提供可视化 UI 交互，实现“原生能力\+快捷操作”的双重体验，解决用户记不住 CLI 命令、skill 及频繁操作终端的痛点，成为 Claude Code CLI 的功能超集工具。

# 二、开发背景与核心痛点

## 2\.1 开发背景

用户已在 Mac 设备上安装 Claude Code CLI，且安装了多个 skill，但日常使用中面临操作繁琐的问题，希望通过可视化 UI 简化操作，同时保留 CLI 原生功能，实现更高效的 Claude Code 交互体验。

## 2\.2 核心痛点

- 操作繁琐：需通过命令行输入指令开启新对话，无法快速调用常用任务；

- 记忆负担：安装的 skill 数量较多，难以全部记住，调用时需反复回忆；

- 交互割裂：启动对话后，输入输出均在终端显示，无法在统一 UI 内完成全流程交互；

- 扩展性不足：常用任务无法预设，每次使用需重复输入相同指令，效率低下。

# 三、核心需求（用户明确提出\+延伸优化）

## 3\.1 基础核心需求

1. 兼容原生：\`claude\-hunter\` 需完全兼容 Claude Code CLI 所有原生功能，输入框支持 CLI 原有特殊符号（\`/\` \`:\` \`@\` 等）、skill 调用、命令触发，交互逻辑与终端 CLI 完全一致，不阉割任何原生能力；

2. 可视化 UI：提供简洁易用的图形界面，所有输入、输出、交互均在 UI 内完成，不弹出 Mac 终端窗口；

3. 快捷交互：支持将常用任务做成配置文件（JSON/MD 格式），通过 UI 按钮点击实现快捷调用，无需手动输入指令；

4. 多轮对话：在 UI 内支持与 Claude Code 多轮沟通，可直接回答 Claude 的提问、补充知识，实现完整对话流程；

5. 智能补全：输入框支持 \`:\`（skill 触发）、\`/\`（命令触发）时自动弹出下拉补全菜单，模拟终端 CLI 的提示体验，降低记忆负担。

## 3\.2 延伸优化需求（基于用户痛点补充）

1. 独立 App 打包：支持打包为 Mac 独立桌面 App，双击即可启动，无需通过终端输入命令运行 Python 脚本；

2. 自动扫描 skill：自动扫描用户本地已安装的所有 Claude Code skill，无需手动配置，自动加载到智能补全列表和快捷菜单；

3. 自定义配置：支持用户自定义任务配置（新增/修改/删除 JSON/MD 任务文件）、UI 样式（按钮颜色、字体等）；

4. 对话管理：支持清空当前对话、重启对话，后续可扩展对话保存、历史记录查询功能；

5. 快捷输入按钮：在 UI 内设置常用操作快捷按钮（如解释代码、找 Bug、重构代码），点击自动填充输入框，提升操作效率。

# 四、详细功能设计（按模块划分）

## 4\.1 整体界面设计

工具界面采用左右分栏布局，兼顾快捷操作与对话交互，整体风格简洁、易用，适配 Mac 系统视觉体验：

- 左侧（快捷面板，宽度固定）：包含新对话按钮、清空输出按钮、快速技能菜单、自定义任务按钮，支持滚动查看（适配多任务场景）；

- 右侧（对话区域，可拉伸）：包含对话输出显示框、快捷输入按钮行、输入框，所有输入输出均在此区域完成。

## 4\.2 原生 CLI 兼容模块（核心基础）

该模块是工具的核心，确保 \`claude\-hunter\` 完全兼容 Claude Code CLI 所有功能，不改变用户原有使用习惯：

1. 底层接管：通过 Python 子进程（QProcess）直接启动 Claude Code CLI 原生内核，接管 stdin（输入）、stdout（输出）、stderr（错误），实现全流程内嵌；

2. 输入兼容：输入框支持所有 CLI 原生操作，包括：
        

    - 输入 \`:\` 触发 skill 调用（如 \`:code\-review\` \`:python\-dev\`）；

    - 输入 \`/\` 触发 CLI 命令（如 \`/help\` \`/new\` \`/clear\`）；

    - 输入 \`@\` 附加文件（如 \`@test\.py\`）；

    - 支持 CLI 原有快捷键、自动提示逻辑，与终端操作完全一致。

3. 输出兼容：Claude Code 的所有输出（回答、提示、错误信息）均实时显示在 UI 输出框，保留原生格式、颜色（优化为深色模式，提升可读性），支持滚动、复制。

## 4\.3 智能补全模块

解决用户记不住 skill 和命令的痛点，模拟终端 CLI 提示体验，提升输入效率：

1. 触发逻辑：输入框输入 \`:\` 时，自动弹出 skill 补全下拉菜单；输入 \`/\` 时，自动弹出命令补全下拉菜单；输入其他内容时，隐藏补全菜单；

2. 补全功能：支持模糊匹配（输入前缀即可匹配相关 skill/命令），鼠标点击补全项可直接填充到输入框，无需手动输入完整内容；

3. 列表更新：支持手动配置 skill/命令列表，后续可扩展“自动扫描本地 skill”功能，自动同步用户已安装的所有 skill，无需手动维护。

# 五、技术实现思路

## 5\.1 开发环境

- 开发语言：Python 3\.8\+（兼容 Mac 系统主流 Python 版本）；

- UI 框架：PyQt6（跨平台、轻量，适配 Mac 系统视觉风格，支持丰富的控件和交互）；

- 打包工具：pyinstaller（支持将 Python 脚本打包为 Mac 独立 App）；

- 依赖库：PyQt6（UI 开发）、subprocess（子进程管理，可选，用于兼容旧版交互）。

## 5\.2 核心技术实现

1. 原生 CLI 接管：使用 QProcess 类启动 Claude Code CLI 子进程，通过 \`readyRead\` 信号监听输出，通过 \`write\` 方法发送输入，实现全流程内嵌；

2. 智能补全：自定义下拉列表控件（AutoCompleteList），监听输入框文本变化，根据输入前缀匹配 skill/命令列表，实现动态补全；

3. 任务配置加载：使用 Path 类扫描 \`tasks/\` 文件夹，分别解析 JSON/MD 文件，提取任务名称和预设指令，动态生成快捷按钮；

4. UI 布局：使用 QSplitter、QVBoxLayout、QHBoxLayout 实现灵活布局，支持左右分栏拉伸，适配不同屏幕尺寸；

5. App 打包：使用 pyinstaller 命令打包，指定 \`\-w\` 参数隐藏终端，\`\-n\` 参数设置 App 名称，生成独立可执行文件。

## 5\.3 关键问题解决

- CLI 原生兼容：通过直接调用 Claude Code CLI 子进程，而非模拟 CLI 逻辑，确保所有原生功能（特殊符号、skill、命令）完全兼容，避免出现交互差异；

- 输入输出同步：使用信号槽机制（pyqtSignal）监听子进程输出，实时更新 UI 输出框，确保输入输出无延迟、无遗漏；

- 智能补全体验：优化补全菜单的触发时机和显示位置，确保与终端 CLI 提示体验一致，同时支持模糊匹配，提升使用便捷性；

- 独立 App 兼容性：打包时自动打包依赖库，确保用户无需安装 Python 环境和相关依赖，双击即可正常启动。

文件路径：\`tasks/代码审查\.json\`

```json

  "cli_args": "--skill code-review --model claude-3.5-sonnet",
  "prompt": "请帮我严格审查这段代码，找出bug、性能问题、安全风险、可读性问题，并给出修复后的完整代码。"
```

```markdown

2. 修复方案
3. 改好的完整代码
```

2. 独立 App 版：将 \`claude\-hunter\.app\` 拖入 Mac 应用程序文件夹，双击即可启动，无需终端操作。
